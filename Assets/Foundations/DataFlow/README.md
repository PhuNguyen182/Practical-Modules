## üìñ Overview

This tool provides **complete control over your PlayerPrefs data** with an intuitive visual interface. It automatically discovers all your `[Serializable]` classes that implement `IGameData` and lets you **edit them like Unity Inspector fields**.

### ‚≠ê Key Features
- üîç **Smart JSON Detection**: Automatically scans PlayerPrefs for existing JSON data
- ‚ú® **Auto-Discovery**: Finds all `IGameData` implementations and matches with saved data
- üé® **Beautiful UI**: Modern UI Toolkit interface with dark theme
- üìù **Live Editing**: Edit data values like Unity Inspector fields
- üíæ **Granular Control**: Load/save individual data types or all at once
- üîß **Flexible Key Matching**: Finds data using multiple naming patterns
- üìÑ **Enhanced JSON Handling**: Robust serialization/deserialization with error handling
- üîí **Safe Operations**: Validation and confirmation dialogs for all operations
- üöÄ **Cross-Platform**: Works on Windows, Mac, and Linux with platform-specific optimizations

### üéØ Use Cases
- üì± **Debug player progress** during development
- üéÆ **Test different game states** without playing through content
- üåü **Reset specific data** without losing everything

### üìÅ Data Storage Pattern

#### PlayerPrefs Key Detection

The tool intelligently scans for JSON data using **multiple key patterns**:

**Primary Pattern:**
```
GameData_{TypeName}
```

**Alternative Patterns Detected:**
- `TypeName` (e.g., `PlayerProgressData`)
- `Data_TypeName` (e.g., `Data_PlayerProgressData`)
- `Save_TypeName` (e.g., `Save_PlayerProgressData`)
- `Player_TypeName` (e.g., `Player_PlayerProgressData`)
- `TypeNameData` (e.g., `PlayerProgressData`)
- `TypeNameConfig` (e.g., `PlayerProgressConfig`)

**Smart Matching:**
- ‚úÖ **JSON Validation**: Only processes keys containing valid JSON
- ‚úÖ **Type Compatibility**: Attempts deserialization to ensure data matches
- ‚úÖ **Fallback Scanning**: Scans all PlayerPrefs keys as last resort
- ‚úÖ **Cross-Platform**: Works on Windows (Registry), Mac (plist), Linux

**Examples:**
```csharp
// All these keys would be detected for PlayerProgressData:
PlayerPrefs.SetString("GameData_PlayerProgressData", json);  // ‚úÖ Primary
PlayerPrefs.SetString("PlayerProgressData", json);           // ‚úÖ Direct
PlayerPrefs.SetString("Data_PlayerProgressData", json);      // ‚úÖ Prefixed
PlayerPrefs.SetString("PlayerProgress", json);               // ‚úÖ Shortened
```

This flexible approach works with **any existing save system** without requiring code changes!
